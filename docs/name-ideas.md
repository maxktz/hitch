# Name Ideas

Strongest short-name candidates for this CLI:

## Top Picks

- `muxi` - Short, tmux-adjacent, and friendly. It hints at the implementation without spelling out "agent" or "terminal."
- `peek` - Clear product metaphor: agents peek at running terminals before acting. Very memorable, but likely harder to own because it is generic.
- `panes` - Boring in a good way. Directly describes what the tool inspects and feels natural as a CLI command.
- `scope` - Suggests giving agents broader situational awareness over the local dev environment.
- `termpeek` - More explicit than `peek`, still much shorter than `muxi`, and easy to understand from the name alone.

## Current Favorite

`muxi` is the strongest overall if the product should keep a subtle tmux connection.

`peek` is strongest if the product should feel agent-first and implementation-agnostic.
